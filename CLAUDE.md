# Claude Code Instructions

## プロジェクト概要
実験協力アプリケーション（Flutter/Firebase）

## 開発方針
- **完全自律開発モード**: すべての編集を許可なく実行
- エラーが発生した場合は自動的に修正を試みる
- テストコマンドがある場合は自動実行
- リファクタリングや最適化を積極的に行う

## 自動実行コマンド
```bash
# コード変更後は必ず実行
flutter analyze
dart fix --apply

# ビルドテスト
flutter build web --no-tree-shake-icons
```

## コーディング規約
- Dartの標準的な命名規則に従う
- 不要なコメントは削除
- エラーハンドリングを適切に実装
- パフォーマンスを考慮した実装

## Firebase関連
- Firestoreのセキュリティルールは慎重に変更
- インデックスは必要に応じて自動追加
- 課金に影響する変更は最小限に

## UI/UX
- Material Design 3に準拠
- レスポンシブデザインを維持
- アクセシビリティを考慮

## 許可事項
- ✅ すべてのコード編集
- ✅ ファイルの作成・削除
- ✅ パッケージの追加・更新
- ✅ リファクタリング
- ✅ パフォーマンス最適化
- ✅ バグ修正
- ✅ 機能追加・改善

## 禁止事項
- ❌ 本番環境への直接デプロイ（firebase deployは開発環境のみ）
- ❌ 機密情報のハードコーディング
- ❌ 破壊的なデータベース操作

## デバッグ時の対応
エラーが発生した場合は以下の順序で対応：
1. エラーログを確認
2. 関連ファイルを読み込んで原因特定
3. 修正を実装
4. テストコマンドで確認
5. 問題が解決するまで繰り返す

## その他
- 変更は積極的かつ大胆に
- パフォーマンスとコード品質を常に意識
- ユーザビリティの向上を最優先
